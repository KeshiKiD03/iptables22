<div><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Claro, aquí tienes el contenido organizado y mejorado en formato Markdown para facilitar su lectura y comprensión:</p><hr><h1 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-capitulo--iptables-firewall-clasico-de-linux">🔥 CAPÍTULO — IPTABLES (Firewall clásico de Linux)</h1><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Control total del tráfico de red a nivel de kernel.</p><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-1-que-es-iptables">🧱 1. ¿Qué es iptables?</h2><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x"><code>iptables</code> es una herramienta para gestionar las reglas del firewall del kernel de Linux (Netfilter).</p><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Permite:</p><ul><li>Permitir tráfico</li><li>Bloquear tráfico</li><li>Redirigir puertos</li><li>Hacer NAT</li><li>Filtrar paquetes por IP, puerto, protocolo, interfaz…</li></ul><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x"><strong>Idea clave:</strong> iptables controla qué entra, qué sale y qué se reenvía en tu máquina.</p><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-2-las-3-tablas-principales">🧩 2. Las 3 tablas principales</h2><ul><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">🟦 <strong>filter</strong> (tabla por defecto)<br>Controla el tráfico normal.<br>Cadenas:</p><ul><li>INPUT → tráfico que entra al sistema</li><li>OUTPUT → tráfico que sale del sistema</li><li>FORWARD → tráfico que pasa a través del sistema (routers)</li></ul></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">🟩 <strong>nat</strong><br>Traducción de direcciones (NAT).<br>Cadenas: PREROUTING, POSTROUTING, OUTPUT</p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">🟧 <strong>mangle</strong><br>Modifica paquetes (TTL, QoS, marcas).</p></li></ul><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x"><strong>Idea clave:</strong> 90% del trabajo se hace en filter y nat.</p><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-3-cadenas-y-politicas-por-defecto">🧱 3. Cadenas y políticas por defecto</h2><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Ejemplo de políticas:</p><pre><code class="lang-bash">iptables -P INPUT DROP
iptables -P OUTPUT ACCEPT
iptables -P FORWARD DROP
</code></pre>
<p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x"><strong>Idea clave:</strong> Política DROP + reglas explícitas = firewall seguro.</p><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-4-comandos-esenciales">🛠️ 4. Comandos esenciales</h2><ul><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Ver reglas:<br><code>iptables -L -n -v</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Añadir regla:<br><code>iptables -A INPUT -p tcp --dport 22 -j ACCEPT</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Eliminar regla:<br><code>iptables -D INPUT 1</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Guardar reglas:<br>Debian/Ubuntu:<br><code>iptables-save &amp;gt; /etc/iptables/rules.v4</code><br>CentOS/RHEL:<br><code>service iptables save</code></p></li></ul><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x"><strong>Idea clave:</strong> si no guardas las reglas, se pierden al reiniciar.</p><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-5-ejemplos-practicos">🧬 5. Ejemplos prácticos</h2><ul><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Permitir SSH:<br><code>iptables -A INPUT -p tcp --dport 22 -j ACCEPT</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Permitir HTTP/HTTPS:<br><code>iptables -A INPUT -p tcp -m multiport --dports 80,443 -j ACCEPT</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Bloquear IP:<br><code>iptables -A INPUT -s 192.168.1.50 -j DROP</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Permitir solo una IP para SSH:</p><pre><code class="lang-bash">iptables -A INPUT -p tcp --dport 22 -s 203.0.113.10 -j ACCEPT

</code></pre>
</li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Permitir ping:<br><code>iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Bloquear ping:<br><code>iptables -A INPUT -p icmp --icmp-type echo-request -j DROP</code></p></li></ul><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-6-nat-con-iptables-muy-importante">🌐 6. NAT con iptables (muy importante)</h2><ul><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Masquerade (salida a Internet):<br><code>iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE</code></p></li><li><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Port forwarding (redirigir puerto 80 externo → 8080 interno):</p><pre><code class="lang-bash">iptables -t nat -A PREROUTING -i eth0 -p tcp --dport 80 -j DNAT --to-destination 192.168.1.10:8080

Orden de las reglas (crítico)</h2><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">iptables evalúa reglas de arriba hacia abajo.</p><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Ejemplo incorrecto:</p><pre><code class="lang-bash">iptables -A INPUT -j DROP

</code></pre><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Resultado: SSH bloqueado porque la regla DROP se ejecuta primero.</p><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x"><strong>Idea clave:</strong> el orden de las reglas lo es todo.</p><hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-8-buenas-practicas-de-seguridad">🛡️ 8. Buenas prácticas de seguridad</h2><ul><li>Política por defecto: DROP</li><li>Permitir solo lo necesario</li><li>Registrar intentos sospechosos</li><li>No permitir SSH desde cualquier sitio</li><li>Usar <code>-m state --state ESTABLISHED,RELATED</code></li><li>Guardar reglas</li><li>Documentar reglas complejas</li></ul><p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">Ejemplo seguro:</p><pre><code class="lang-bash">iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
</code></pre>
<hr><h2 class="MuiTypography-root MuiTypography-headingMedium css-qp7i24" id="-9-ejemplo-de-firewall-completo-tipico">🧪 9. Ejemplo de firewall completo (típico)</h2><pre><code class="lang-bash">iptables -F

</code></pre>

</code></pre>
<p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">&gt; Se limpian todas las reglas actuales y se establece que por defecto todo sea aceptado para evitar bloqueos inesperados.</p><hr><h3 class="MuiTypography-root MuiTypography-headingSmall css-1akoz7" id="2-nat-para-redes-internas-neta-y-netb">2. NAT para redes internas NetA y NetB</h3><pre><code class="lang-bash">iptables -t nat -A POSTROUTING -s 172.20.0.0/16 -o eno1 -j MASQUERADE
iptables -t nat -A POSTROUTING -s 172.30.0.0/16 -o eno1 -j MASQUERADE
</code></pre>
<p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">&gt; Se ocultan las IPs internas de las redes NetA y NetB detrás de la IP del router para acceder a Internet.</p><hr><h3 class="MuiTypography-root MuiTypography-headingSmall css-1akoz7" id="3-restringir-conexiones-ssh-al-host-amfitrio">3. iptables -A INPUT -s 192.168.2.1 -p tcp --dport 22 -i eno1 -j ACCEPT

# Permitir respuestas SSH a 192.168.2.1
iptables -A OUTPUT -d 192.168.2.1 -p tcp --sport 22 -o eno1 -m state --state ESTABLISHED,RELATED -j ACCEPT

# Bloquear resto conexiones SSH entrantes
iptables -A INPUT -p tcp --dport 22 -i eno1 -j DROP
</code></pre>
<p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">&gt; Solo permitimos conexiones SSH entrantes desde una IP específica y bloqueamos el resto.</p><hr><h3 class="MuiTypography-root MuiTypography-headingSmall css-1akoz7" id="4-ssh-saliente-solo-a-red-1921682024">4. SSH saliente solo a red 192.168.2.0/24</h3><pre><code class="lang-bash"># Permitir SSH saliente solo a 192.168.2.0/24
iptables -A OUTPUT -d 192.168.2.0/24 -p tcp --dport 22 -o eno1 -j ACCEPT

# Permitir respuestas SSH entrantes desde 192.168.2.0/24
iptables -A INPUT -s 192.168.2.0/24 -p tcp --sport 22 -i eno1 -m state --state ESTABLISHED,RELATED -j ACCEPT

# Bloquear resto de conexiones SSH salientes
iptables -A OUTPUT -p tcp --dport 22 -o eno1 -j DROP
</code></pre>
<p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">&gt; Solo permitimos que el host haga conexiones SSH a la red 192.168.2.0/24 y bloqueamos el resto.</p><hr><h3 class="MuiTypography-root MuiTypography-headingSmall css-1akoz7" id="5-restriccion-de-acceso-entre-neta-y-netb">5. Restricción de acceso entre NetA y NetB</h3><pre><code class="lang-bash"># Permitir NetA acceder solo a b1 en NetB
iptables -A FORWARD -s 172.20.0.0/16 -d 172.30.0.2 -j ACCEPT

</code></pre>
<p class="MuiTypography-root MuiTypography-bodyMedium MuiTypography-paragraph css-14g3j2x">&gt; Solo se permite que la red NetA acceda a un host específico (b1) en NetB, el resto queda bloqueado.</p><hr><h3 class="MuiTypography-root MuiTypography-headingSmall css-1akoz7" id="6-redirigir-puerto-80-al-servidor-web-s1-en-dmz">6. </code></pre>
